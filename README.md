# Локальный Live Share для Python

Браузерный live-редактор для `liveshare.py` между двумя компьютерами в одной локальной сети.

## Требования

- Python 3.8+
- `pip install -r requirements.txt`

## Состав

- `sync_server.py` - сервер учителя, который отдает web UI и синхронизирует текст
- `sync_client.py` - локальный bridge на каждой машине, который записывает `liveshare.py`
- `liveshare.py` - общий Python-файл, который каждый запускает локально

## Порты

- `8000` - web UI
- `5678` - WebSocket-синхронизация
- `8765` - локальный bridge на каждой машине

## Запуск

На компьютере учителя:

```bash
python sync_server.py
python sync_client.py
```

После этого откройте в браузере:

```text
http://127.0.0.1:8000
```

На компьютере ученика:

```bash
python sync_client.py
```

После этого откройте в браузере:

```text
http://192.168.0.102:8000
```

Вместо `192.168.0.102` используйте IP компьютера учителя.

## Как это работает

1. Оба участника открывают одну и ту же web-страницу редактора.
2. При наборе текста страница отправляет весь текущий код на сервер с debounce около `75` мс.
3. Сервер сохраняет новую версию и мгновенно рассылает ее второму участнику.
4. Браузер на каждой машине пишет полученный текст в локальный `liveshare.py` через `sync_client.py`.
5. Каждый запускает `python liveshare.py` локально у себя.

## Ограничения

- Используется `last write wins`.
- Синхронизируется только `liveshare.py`.
- Для записи файла браузеру нужен запущенный `sync_client.py` на этой машине.
